Networking & The Internet Terminology

Loren Lemarr

TERM DEFINITION
Network Classifications
LAN A local area network normally consists of a collection of computers in a single buildingg or building complex.
MAN A metropolitan area network is a network of intermediate size, such as one spanning a local community.
WAN A wide area network links machines over a greater distance. (The internet)
Network Models
bus network Network topology in which the machines are all connecterd to a common communication line called a bus.
star network Network topology in which a central machine, called the access point, serves as a focal point around which all communication is coordinated.
Protocols
CSMA/CD Ethernet Carrier Sense, Multiple Access with Collision Detection controls the right to transmit messages. The protocols dictates that each message be broadcast to all the machines on the bus. Each machine monitors all the messages but only keeps those addressed to itself. Machines wait till the line is silent to transmit a message and if a collision is detected both machines involved pause then try again a after a random length of time.
CSMA/CA Wifi CSMA/CD is not compatible with wireless star networks, since a machine may not be able to detect that its transmissions are colliding with another machine's transmissions. The result is that wireless networks adopt the policy of trying to avoid collisions rather than detect them.
Combining networks
repeater A device that passes signals back and forth between the two original buses without considering the means of the signal.
bridge Connects two buses like a repeater, but it does not neccessarily pass all messages across the connection. Instead, it looks at the destination address that accompanies each message and forwards a message across the connection only when that message is destined for a computer on the other side.
switch A switch is essentially a bridge with multiple connections, allowing it to connect several buses rather than just two. As in the case of a bridge, a switch considers the destination addresses of all messages and forwards only those messages destined for other spokes.
router Handles the connection between networks to form an internet. They are special purpose computers used for forwarding messages. They provide links between networks while allowing each network to maintain its unique internal characteristics.
gateway The “point” at which one network is linked to an internet. They serve as a passageway between the network and the outside world.
Client/Server model
P2P: peer-to-peer The peer-to-peer model involves processes that provide ser- vice to and receive service from each other. It usually involves processes that execute on a temporary basis.
distributed system Software systems that consist of software units that execute as processes on different computers.
cloud computing Huge pools of shared computers on the network that can be allocated for use by clients as needed, is the latest trend in distributed systems.
Internet Addressing
Tier 1 Tier-1 ISPs that consist of very high-speed, high-capacity, international WANs. These networks are thought of as the backbone of the Internet. They are typically operated by large companies that are in the communications business.
Tier 2 Compared to tier-1, tier-2 ISPs tend to be more regional in scope and less potent in their capabilities. These networks tend to be operated by companies in the communications business.
Tier 3 Also called an access ISP it is essentially an independent internet, sometimes called an intranet, operated by a single authority that is in the business of supplying Internet access to individual homes and businesses.
IP address
ICANN The Internet Corporation for Assigned Names and Numbers is non-profit corporation established to coordinate the Internet's operation. Awards blocks of consecutively numbered IP addresses to ISPs
DNS the software being used must be able to convert mnemonic addresses into an IP address before transmitting the message. This conversion is performed with the aid of numerous servers called name servers. Collectively, these name servers are used as an Internet-wide directory system known as the domain name system (DNS).
TLD: top-level domain Suffix that follows the period that reflects the domains classification. Includes .edu for educational use, .com for commercial institutions, .gov for U.S. government institutions, .org for nonprofit organizations, .mil for military, and .net, which was originally intended for ISPs but is now used on a much broader scale.
Country-code TLDs Two-letter TLDs for each specific country (called country-code TLDs) such as .au for Australia and .ca for Canada.
Internet Protocols
FTP File Transfer Protocol (FTP), or an application for accessing another com- puter from a great distance used the Telnet protocol, or later, the Secure Shell (SSH) protocol.
HTTP Hypertext Transfer Protocol transfers hypertext documents between browsers and webservers
SMTP SMTP (Simple Mail Transfer Protocol) defines a way that two computers on the network may interact when transmitting an email message from one host to the other.
World Wide Web
URL In order to locate and retrieve documents on the Web, each document is given a unique address called a Uniform Resource Locator (URL). Each URL contains the information needed by a browser to contact the proper server and request the desired document.
client-side activities Activities that are performed by a client(such as a browser).
server-side activities Activities that are performed by a server(such as a webserver).
Internet four-layer software
aplication layer The application layer uses the transport layer to send and receive messages over the Internet in much the same way that you would use a shipping company to send and receive packages. It is its responsibility provide an address that is compatible with the Internet infrastructure.
transport layer Accepts messages from the application layer and to ensure that the messages are properly formatted for transmission over the Internet. Toward this latter goal, the transport layer divides long messages into small segments, which are transmitted over the Internet one segment at a time. The transport layer adds sequence numbers to the small segments it pro- duces so that the segments can be reassembled at the message’s destination.
network layer It is the network layer's job to decide in which direction a packet should be sent at each step along the packet’s path through the Internet. The network layer is in charge of maintaining the router's forwarding table and using that table to determine the direction in which to forward packets.
link layer The link layer at the router is in charge of receiving and transmitting the packets.
Internet is:
packet-switching network Packet switching is the transfer of small pieces of data across various networks.
fault tolerant Fault tolerance refers to the ability of a system (computer, network, cloud cluster, etc.) to continue operating without interruption when one or more of its components fail.
redundant Network redundancy is the process of providing multiple paths for traffic so that data can keep flowing even in the event of a failure. Put simply: more redundancy equals more reliability. It also helps with distributed site management. The idea is that if one device fails, another can automatically take over.
Cybersecurity
malware Software that is specifically designed to disrupt, damage, or gain unauthorized access to a computer system.
virus A computer program that can copy itself and infect a computer without permission or knowledge of the user. A virus might corrupt or delete data on a computer, use e-mail programs to spread itself to other computers, or even erase everything on a hard disk.
worm A computer worm is a type of malware whose primary function is to self-replicate and infect other computers while remaining active on infected systems. A computer worm duplicates itself to spread to uninfected computers.
Trojan horse A computer program that appears to have a useful function, but also has a hidden and potentially malicious function that evades security mechanisms, sometimes by exploiting legitimate authorizations of a system entity that invokes the program.
spyware Spyware is malicious software that enters a user's computer, gathers data from the device and user, and sends it to third parties without their consent. A commonly accepted spyware definition is a strand of malware designed to access and damage a device without the user's consent.
phishing A technique for attempting to acquire sensitive data, such as bank account numbers, through a fraudulent solicitation in email or on a web site, in which the perpetrator masquerades as a legitimate business or reputable person.
DDOS attack DDoS Attack means "Distributed Denial-of-Service (DDoS) Attack" and it is a cybercrime in which the attacker floods a server with internet traffic to prevent users from accessing connected online services and sites.
Protection
firewall A primary prevention technique that filters traffic passing through a point in the network.
spam filter which are firewalls designed to block unwanted email. Many spam filters use rather sophisticated techniques to distinguish between desirable email and spam.
proxy server A proxy server is a software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server. Without a proxy server, a client communicates directly with a server, meaning that the server has an opportunity to learn a certain amount about the client.
antivirus software used to detect and remove the presence of known viruses and other infections. (Actually, antivirus software represents a broad class of software products, each designed to detect and remove a specific type of infection. For example, whereas many products spe- cialize in virus control, others specialize in spyware protection.)
cryptography Cryptography is the study of sending and receiving messages securely in the presence of adversaries. Cryptographic tools such as encryption can be used to trans- form a message into a coded version such that even if the encrypted data fall into unscrupulous hands, the encoded information will remain confidential.
public key encryption involves techniques by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
